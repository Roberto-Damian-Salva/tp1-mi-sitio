# Reflexión sobre Metodologías Ágiles

### (a) ¿Qué diferencia hay entre Scrum y Kanban? ¿Cuál te parece más útil para un proyecto individual y por qué?
La diferencia principal es que Scrum usa tiempos fijos (sprints) y roles , mientras que Kanban es un flujo continuo sin interrupciones. Para un proyecto individual prefiero Kanban porque es más flexible y permite ver visualmente qué tareas faltan.

### (b) ¿Qué es un MVP (Producto Mínimo Viable) y cómo lo aplicarías si te piden hacer un sitio web para un negocio local?
Un MVP es la versión más simple de un producto que ya funciona para el usuario. Para un negocio local, mi MVP sería una página con el menú/servicios, horario y un botón de WhatsApp para pedidos, dejando funciones complejas como pagos online para más adelante.

### (c) ¿Cómo te ayudó el tablero Kanban a organizar las tareas de este TP? Menciona un ejemplo concreto.
Me ayudó a no olvidarme de pasos técnicos importantes. Por ejemplo, me sirvió para separar el maquetado de los estilos de validación, asegurándome de que el HTML funcionara bien antes de ponerme a diseñar los colores del CSS.
